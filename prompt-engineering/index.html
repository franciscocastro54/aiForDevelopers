Trabajando con IA

Prompt engineering

Estructura, patrones y técnicas para escribir prompts que producen resultados consistentes.

Un prompt mal escrito no es solo ineficiente: produce código que parece correcto pero resuelve un problema diferente al que tenías. La diferencia entre un prompt bueno y uno malo puede ser la diferencia entre 5 minutos de trabajo y 2 horas de debugging de código generado.

Anatomía de un prompt efectivo

El patrón Rol · Contexto · Tarea · Restricciones · Formato cubre la mayoría de los casos:

  • Rol: qué tipo de experto quieres que sea. "Eres un developer senior especializado en performance de bases de datos."
  • Contexto: qué sabe sobre tu situación. Stack, restricciones, codebase relevante.
  • Tarea: qué quieres que haga, específico y accionable.
  • Restricciones: qué no debe hacer. "No uses librerías externas", "mantén la API pública igual".
  • Formato: cómo quieres la respuesta. "Solo código, sin explicación", "primero el análisis, después el código".

Iteración de prompts

Los prompts se refinan, no se escriben perfectos la primera vez. Cuando el resultado no es el esperado, el error suele estar en uno de estos lugares:

  • Faltó contexto que tú tienes obvio pero la IA no sabe
  • La restricción que importaba no estaba explícita
  • El formato pedido no fue suficientemente específico
  • La tarea era demasiado grande para un solo prompt

Antes de reformular todo, intentá agregar solo lo que faltó. Un prompt que funciona vale más que uno que parece elegante.

💡 Meta-prompt

Si no sabes cómo formular tu pregunta, empieza con: "¿Qué información necesitarías de mi parte para ayudarme con X?" La IA te va a decir exactamente qué contexto le falta.
