Trabajando con IA

Optimización de tokens

Context windows, costos y técnicas para hacer más con menos tokens.

El token es la unidad mínima de costo en una API de IA. Cada carácter que envías y recibes se cobra. En proyectos grandes con equipos usando IA continuamente, el costo puede escalar rápidamente. Pero optimizar tokens no es solo cuestión de dinero: un contexto bien diseñado produce respuestas más precisas que uno inflado.

Cómo funciona el context window

El contexto es todo lo que el modelo "ve" en un momento dado: el system prompt, la conversación hasta ahora, los archivos que adjuntaste, y el mensaje actual. Los modelos actuales tienen ventanas de 200k a 1M tokens, pero eso no significa que debas llenarlos.

  • Más contexto = más costo de input
  • Más contexto = más latencia
  • Contexto irrelevante puede "distraer" al modelo y bajar la calidad de la respuesta

Técnicas de reducción

1. Archivos de reglas persistentes (CLAUDE.md / .cursorrules)

En vez de repetir en cada conversación "usa TypeScript strict, no uses any, los tests van en __tests__/", guardas esas reglas en un archivo que el cliente de IA carga automáticamente. Es el ROI más alto de todas las optimizaciones.

2. Caché de prompts

Los proveedores principales (Anthropic, OpenAI) ofrecen caché de prompt: si el inicio de tu contexto no cambia entre requests, lo cobran a una fracción del precio. Esto es especialmente útil cuando tienes un system prompt largo o incluyes documentación de referencia.

  • En Anthropic: el caché aplica automáticamente para bloques mayores a ~1000 tokens que no cambian
  • Discount típico: 90% en tokens cacheados
  • Estrategia: poné el contenido estático (reglas, documentación) al inicio del contexto, el contenido dinámico al final

3. Modelo correcto para cada tarea

TareaModelo recomendadoPor qué
Razonamiento complejo, arquitecturaOpus / GPT-4oMayor capacidad de razonamiento multi-paso
Generación de código estándarSonnet / GPT-4o-miniBalance costo/calidad óptimo
Boilerplate, renombrado, formateoHaiku / GPT-3.5Tareas mecánicas, no necesitan inteligencia alta
Embeddings, clasificaciónModelos especializadosMucho más baratos que los de chat

4. Contexto mínimo suficiente

Enviá solo lo que la IA necesita para la tarea específica. Si estás arreglando un bug en auth/login.ts, no enviés todo el directorio. Si estás refactorizando una función, enviá esa función y sus tipos, no el archivo completo.

5. Compresión de conversación

Las sesiones largas acumulan contexto. Los clientes como Claude Code hacen compresión automática, pero en APIs directas eres tú quien maneja la historia. Estrategia: cuando la conversación supera las 50-60 interacciones, pedí un resumen del estado actual y empezá una sesión nueva con ese resumen como contexto inicial.

ℹ️ Referencia de precios (aproximada, verificar en el sitio del proveedor)

Anthropic Claude: Haiku ~$0.80/MTok input, Sonnet ~$3/MTok, Opus ~$15/MTok. Con caché de prompt, los tokens cacheados cuestan ~10% del precio base. Un millón de tokens de input equivale aproximadamente a 750,000 palabras.
