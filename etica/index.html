Estrategia

Ética e IP

Propiedad intelectual del código generado, licencias, responsabilidad y disclosure a clientes.

Las preguntas legales alrededor del código generado por IA no están completamente resueltas. Lo que sí está claro es que la incertidumbre no desaparece ignorándola, y que algunos riesgos son reales hoy, no hipotéticos.

Propiedad del código generado

La posición mayoritaria en el mundo legal actualmente: el código generado por IA no tiene copyright automático porque no hay un autor humano. El output no es de la IA (que no puede tener derechos), ni automáticamente tuyo por pedirlo.

Lo que esto implica en la práctica:

  • El código que generaste con IA puede usarse en tu proyecto comercial
  • No podés registrarlo como tuyo si fue generado enteramente por IA
  • Si el código generado incluye fragmentos copiados de código con licencia, esa licencia aplica

Esta área evoluciona rápido. Consultá un abogado para decisiones de alto impacto.

El problema de las licencias copyleft

Los modelos se entrenaron con código de repositorios públicos, incluyendo código GPL, LGPL y otras licencias copyleft. Si el modelo "memorizó" y regurgita fragmentos de ese código, ¿aplica la licencia copyleft a tu proyecto?

La respuesta honesta: es una pregunta abierta en los tribunales. La posición práctica de la mayoría de las empresas: usar herramientas (como GitHub Copilot Duet AI) que tienen protección legal contractual frente a este escenario, o evitar el uso de IA en código que irá bajo licencia restrictiva.

⚠️ Proyectos open source

Si contribuís a un proyecto open source con código generado por IA, verificá las políticas del proyecto. Algunos proyectos importantes (Linux kernel, por ejemplo) tienen posiciones explícitas sobre contribuciones con IA.

Responsabilidad cuando el código falla

Si el código generado por IA causa un incidente en producción —pérdida de datos, brecha de seguridad, falla de servicio— la responsabilidad sigue siendo del desarrollador y la empresa. "Lo generó la IA" no es una defensa válida frente a clientes, reguladores ni en contratos.

Esto refuerza la práctica de entender y revisar todo el código generado antes de deployar: no es solo buena práctica de ingeniería, es responsabilidad profesional.

Disclosure a clientes

No hay obligación legal universal de informarle a clientes que usaste IA para desarrollar su software. Las excepciones:

  • El contrato lo especifica explícitamente
  • El proyecto está en una industria regulada que tiene reglas sobre herramientas de desarrollo (poco común, pero existe)
  • Enviaste datos del cliente a la IA (en cuyo caso puede haber obligación de notificación por privacidad)

Más allá de lo legal: si el cliente pregunta directamente, ser honesto sobre el uso de IA es lo correcto profesionalmente. La mayoría de los clientes no tiene problemas con el uso de herramientas de IA siempre que la calidad se mantenga.

Uso de datos del cliente con IA

Esta es el área de mayor riesgo real hoy. Si enviás datos del cliente —esquemas de base de datos, logs con comportamiento de usuarios, código que maneja su información personal— a una API de IA pública, podés estar violando:

  • El contrato con el cliente (especialmente si tiene cláusulas de confidencialidad)
  • GDPR si el cliente es europeo y los datos incluyen PII
  • SOC2 o ISO 27001 si tu empresa tiene esas certificaciones
